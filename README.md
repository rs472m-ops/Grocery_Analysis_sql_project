# 🛒 Grocery Store Management System (SQL Project)

## 📌 Project Overview

This project focuses on designing and implementing a **relational database system** for a grocery store using SQL.
It simulates real-world retail operations by managing data related to customers, products, suppliers, employees, and orders.

The project also includes **data analysis queries** to extract meaningful business insights.

---

## 🎯 Objectives

* Design a structured relational database
* Manage data for customers, products, suppliers, and orders
* Perform data retrieval using SQL queries
* Analyze sales trends and customer behavior
* Apply SQL concepts like joins, aggregations, and subqueries
* Generate insights for business decision-making

---

## 🗂️ Database Schema

The project consists of the following tables:

* **supplier** – Stores supplier details
* **categories** – Product categories
* **products** – Product information
* **customers** – Customer details
* **employees** – Employee data
* **orders** – Customer orders
* **order_details** – Order line items

---

## 🔗 Entity Relationships

* One-to-Many: Supplier → Products
* One-to-Many: Categories → Products
* One-to-Many: Customers → Orders
* One-to-Many: Employees → Orders
* One-to-Many: Orders → Order Details
* One-to-Many: Products → Order Details

---

## 📊 Key Analysis Performed

### 👥 Customer Insights

* Total number of customers placing orders
* Top customers by spending
* Average customer purchase value

### 🛒 Product Performance

* Best-selling products
* Revenue generated per product
* Category-wise product performance

### 📈 Sales Trends

* Total number of orders
* Average order value
* Monthly sales and revenue trends

### 🏭 Supplier Analysis

* Supplier contribution to product inventory
* Revenue generated by each supplier

### 👨‍💼 Employee Performance

* Orders handled by employees
* Total sales processed per employee
